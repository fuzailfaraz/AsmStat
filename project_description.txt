AsmStat - Dual Architecture Statistical Engine
==============================================

Project Overview
----------------
AsmStat is a comprehensive dual-architecture statistical engine that bridges the gap between low-level hardware programming and modern, interactive web interfaces. This project demonstrates proficiency in both modern 64-bit Assembly integrated with Python, as well as retro 16-bit DOS Assembly using EMU8086. It showcases a deep understanding of computer architecture, ABI (Application Binary Interface), memory management, full-stack integration, and modern Data Science methodologies.

The project is split into two distinct parts:

Part 1: AsmStat Pro Max (Modern 64-bit & Python Web App)
--------------------------------------------------------
The modern iteration utilizes a high-performance 64-bit Assembly backend combined with a beautiful, glassmorphism-styled Python Streamlit frontend. It serves as a fully-fledged Data Science Analytics Platform.

Key Features:
- 64-bit NASM Backend (`math.asm` & `math_linux.asm`): Core statistical operations (Sum, Mean, Variance, Min, Max, Dot Product, StdDev) written in pure 64-bit x86-64 assembly. Supports both Windows (Microsoft x64 ABI) and Linux (System V AMD64 ABI).
- Python Integration (`app.py`): Uses Python's `ctypes` library to seamlessly load the Assembly DLL/SO and execute native machine code directly from the web app. Features an automated runtime compilation pipeline using `subprocess` (nasm + ld) for Streamlit Cloud deployment.
- Premium UI/UX: Built with Streamlit, featuring a massive, premium dark-mode interface with frosted glassmorphism, animated hover effects, custom Google Fonts ("Outfit"), and a built-in state-based page router.
- Full Data Science Suite: Integrates `plotly` and `pandas` to dynamically generate Bar Charts, Trend Lines, Box Plots, Violin Plots, and ECDFs.
- Advanced Dataset EDA: Supports `.csv` and `.xlsx` uploads with robust encoding fallbacks. Features automated missing value analysis, interactive Pearson correlation heatmaps, multi-variate scatter pairplots, and IQR-based anomaly detection. Includes an "Individual Column Deep-Dive" module.

How to Run (Part 1):
1. Prerequisites: Ensure you have NASM and GCC/ld installed.
2. Compile Assembly Code (Windows): Run the build script `build.bat` to compile `math.asm` into `math.dll`. On Linux, the app handles runtime compilation automatically.
3. Install Dependencies: Install the required Python packages (`pip install -r requirements.txt`).
4. Launch the App: Run `streamlit run app.py`

Part 2: AsmStat Classic (16-bit EMU8086 Edition)
------------------------------------------------
The retro iteration of the project is a standalone, interactive Command Line Interface (CLI) application written entirely in 16-bit DOS Assembly for the EMU8086 emulator. It includes an automated file-sync bridge to push data directly to the modern web UI.

Key Features:
- Pure 16-bit Environment (`AsmStat.asm`): Uses legacy 16-bit registers (AX, BX, CX) and DOS interrupts.
- VGA Memory Mapping: Directly writes to the VGA video memory (0B800h) to colorize the ASCII art and terminal interface without external libraries.
- Dynamic Array Handling: Allows the user to dynamically input array lengths and elements, manipulating memory pointers (SI, DI) in real-time.
- In-Place Sorting: Implements an O(N²) Bubble Sort algorithm purely in Assembly to enable Median calculations.
- File Output Syncing: Writes datasets to `DATA.TXT` directly from Assembly, which is then parsed by the Python UI for immediate cross-platform visualization.

Educational Value & Technical Achievements
------------------------------------------
- Cross-Architecture Expertise: Demonstrates differences between 16-bit segmented real mode and modern 64-bit flat memory models (Windows vs Linux ABIs).
- Foreign Function Interfaces (FFI): Interfacing Python with low-level machine code via dynamic link libraries and C-types.
- CI/CD & Cloud Deployment: Engineered a runtime compilation pipeline to bypass container immutability on Streamlit Cloud, adhering to Linux strict memory protections (`.note.GNU-stack`).
- Hardware Interaction: Using software interrupts to communicate directly with BIOS/DOS for screen rendering and input capture.
